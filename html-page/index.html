HTML

A language which acts as the skeleton to provide the structure and content for a webpage

Best practices

  • Comments: use comments to add a brief note or explanation to the code
  • Accessibility: every image needs a clear and brief description "alt"
  • Formatting: make sure each element is appropriately nested
  • Validate the code: use a HTML validator to check the code for errors

Hints

  • Check for start and end tags
  • Check container levels
  • Check folder pathway
  • IDs: are for a single element and are used in nav links
  • Classes: can be assigned to multiple elements and should have a simple and intuitive name based on either the element or the style that will be applied
    Classes can be grouped based on the style “bold” or element “main-heading”
  • “Alt” + “shift” + “f” automatically indents code and makes it easier to check for closing tags

One & Done

  • Forms
  • Form template
  • Section tags vs div or article tags
  • Avoid showing the first item in a droplist
